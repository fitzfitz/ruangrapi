# RuangRapi Product Brief

## Summary

RuangRapi is an operations SaaS for Indonesian rental property owners and small admin teams managing kos, kontrakan, apartments, or small rental portfolios.

The MVP helps owners keep one reliable source of truth for monthly rental operations: properties, units, tenants, leases, invoices, payments, receipts, manual WhatsApp reminders, utility readings, maintenance tickets, and basic dashboard reporting.

## Positioning

RuangRapi is built for rental operators who already have tenants and need a simpler way to run recurring rental administration.

It is not:

- A property marketplace
- A tenant acquisition platform
- A payment gateway
- A tenant mobile app
- A full accounting system
- A tax reporting system
- An enterprise property management system
- An AI automation product

## Problem

Many small rental owners in Indonesia manage rental operations using WhatsApp, notebooks, spreadsheets, and memory.

Common pain points:

- Unit occupancy is hard to see at a glance.
- Tenant and lease information is scattered.
- Monthly billing is repetitive and easy to miss.
- Payment status is tracked manually.
- Receipts are inconsistent or recreated by hand.
- Reminder messages must be prepared from billing and tenant details.
- Utility readings are tracked separately from monthly billing.
- Maintenance issues are mixed into chat history.
- Monthly expected rent and collected rent are hard to compare quickly.

## Target Customer

Primary customer:

- Indonesian rental owner or small rental admin team managing a small portfolio.

Examples:

- Kos owner with multiple rooms
- Kontrakan owner with several units
- Apartment owner renting several units
- Family business managing rental properties
- Admin helping an owner handle monthly rent collection and tenant follow-up

The MVP assumes a simple organization model: one organization owns the operating data, and owner/admin access remains simple for the first version.

## MVP Value Proposition

RuangRapi helps rental owners answer the operational questions that matter every month:

- Which units are occupied, vacant, under maintenance, or inactive?
- Which tenants have active leases?
- Which invoices are draft, unpaid, partially paid, paid, overdue, or cancelled?
- Which payments have been recorded?
- Which receipts have been issued?
- Which utility readings need to be included in billing?
- Which reminder messages need to be prepared?
- Which maintenance tickets are still open?
- How much rent is expected this month?
- How much rent has actually been collected?

The value is operational clarity, not advanced automation: owners should spend less time reconciling spreadsheets and chat messages, and more time acting on clear rental information.

## MVP Success Criteria

The MVP is successful if a small Indonesian rental owner can manage one month of rental operations in RuangRapi without relying on a spreadsheet as the primary source of truth.

For the MVP, success means the owner can:

1. See property and unit occupancy clearly.
2. Track tenants and active leases.
3. Prepare and monitor monthly invoices.
4. Record payments and issue simple receipts.
5. Prepare manual WhatsApp reminder text.
6. Record utility readings and maintenance tickets.
7. View basic expected-versus-collected rent and open-issue summaries.

## Product Boundaries

RuangRapi should stay focused on small rental operations for the MVP.

Do not expand the MVP into:

- Marketplace listing or tenant acquisition
- Payment gateway processing or automated bank reconciliation
- Tenant mobile app access
- Enterprise permissions or multi-organization membership
- Complex accounting, tax, payroll, or inventory workflows
- WhatsApp Business API automation
- AI-generated workflows or recommendations

Detailed domain rules, data-model decisions, and RLS policy planning live in the dedicated source-of-truth documents and should not be duplicated in this brief.
