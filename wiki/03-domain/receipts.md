# Receipts

Receipts are future modules.

## Current status

Not started.

## Future purpose

Receipts provide proof of payment.

## Known numbering direction

Receipt numbers should be organization-scoped, using a pattern such as:

```txt
RR-{YYYY}-{0001}
```

Example:

```txt
RR-2026-0001
```

## Dependencies

Receipts should come after payments.
